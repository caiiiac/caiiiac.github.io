使用高德地图SDK过程中的各种坑

Jul 30th, 2015 3:36 pm

1. APP内调起高德地图导航

高德文档描写如下:

``` //配置导航参数 MANaviConfig * config = [[MANaviConfig alloc] init]; config.destination = view.annotation.coordinate;//终点坐标，Annotation的坐标 config.appScheme = [self getApplicationScheme];//返回的Scheme，需手动设置 config.appName = [self getApplicationName];//应用名称，需手动设置 config.style = MADrivingStrategyShortest; //若未调起高德地图App,引导用户获取最新版本的 if(![MANavigation openAMapNavigation:config]) { [MANavigation getLatestAMapApp]; } ```

实际情况

  • 2.5.0 SDK使用此方法调用导航无任何反应 [MANavigation openAMapNavigation:config] 返回为成功
  • 2.6.0 SDK查无此类

解决方法

  • 更新SDK为2.6.0 pod 'AMap2DMap', '~> 2.6.0'

代码修改如下:

``` //配置导航参数 MANaviConfig * config = [[MANaviConfig alloc] init]; config.destination = _parkingCoordinate; //终点坐标，Annotation的坐标
config.appScheme = [self getApplicationScheme]; //返回的Scheme，需手动设置 config.appName = [self getApplicationName]; //应用名称，需手动设置 config.strategy = MADrivingStrategyShortest;

//若未调起高德地图App,引导用户获取最新版本的 if(![MAMapURLSearch openAMapNavigation:config]) { [MAMapURLSearch getLatestAMapApp]; } ```

2. - (void)setZoomLevel:(double)newZoomLevel animated:(BOOL)animated; 机制区别

使用情况 - 之前2.5.0版本,直接 [_mapView setZoomLevel:16.f animated:YES]在之后就不需要再次设置,地图就可以一直以配置好的ZoomLevel进行显示 - 更新至2.6.0版本后, 突然发现 [_mapView setZoomLevel:16.f animated:YES]失去了效果

解决方法 - 如果有进行POI搜索,在有搜索结果后进行 [_mapView setZoomLevel:16.f animated:YES] - 没有的话,要延时对ZoomLevel进行设置 代码如下: ``dispatchafter(dispatchtime(DISPATCHTIMENOW, (int64t)(0.5 NSECPERSEC)), dispatchgetmainqueue(), ^{ self.mapView setZoomLevel:14.f animated:YES; });
